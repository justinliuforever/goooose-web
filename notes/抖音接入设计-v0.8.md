# 抖音接入设计(v0.8)

2026-07-16。依据:TikHub 抖音 297 端点实测探索(5 路,~240 次真实调用)、生产 worker IDC 冒烟(16/16 全绿)、全仓平台触点盘点(~90 处)。样本 JSON 在 session scratchpad `douyin/`,硬结论在 memory `project_v08-douyin`。

## 0. 一页结论

抖音接入形态**与小红书同侧**:TikHub 出元数据、worker 直连 CDN 拉媒体、零代理(us-east-1 冒烟实测不封 IDC)。改动分三块:

1. **净新增**:`packages/integrations/src/clients/douyin.ts`(照 xhs.ts 模式,约 450 行)+ 迁移 0033(pgEnum 加值一行)
2. **分支扩展**:worker 两条管线(analyze-channel / monitor-competitors)从 `if(xhs)/else` 重构为显式三路;routers.ts 三个函数同构改造;web UI 约 22 个文件的小分支/文案
3. **P1 特色**(用户已拍板全要):弹幕留人曲线、星图受众画像、平台热点巡视、关键词趋势、垂类达人推荐

## 1. 端点选型(全部实测过)

| 场景 | 端点(REST `/api/v1/douyin/...`) | 单价 | 要点 |
|---|---|---|---|
| 分享文本→账号 | 自建正则抠 URL + 短链自跟 302 → `web/get_sec_user_id` | $0.001 | TikHub 不跟 `v.douyin.com` 重定向、分享口令文本会抛裸异常,必须自建预处理(复用 expandXhsShortLink 模式) |
| 绑定校验/粉丝刷新 | `web/handler_user_profile`(v1) | $0.001 | 错误形状干净(`status_code:2`);一次返回 uid/unique_id/sec_uid 全套 |
| 对标导入元数据 | `web/handler_user_profile_v4` | $0.003 | 含企业认证/IP 属地;假 id 抛裸 `{"error":...}` 要 catch |
| 作品列表 | `app_v3/fetch_user_post_videos` | $0.001 | web 版对活跃号缓存滞后 6 天,弃用;分页 `max_cursor`(=create_time×1000)+`has_more`;置顶 `is_top` 恒在第 1 页且不占 count |
| 单条详情(拆解时现取 play_addr) | `app_v3/fetch_one_video` / `fetch_one_video_by_share_url` | $0.001 | 长短链都吃;web 版 bit_rate 11 档但 P0 用不上高清 |
| 真实播放数回填 | `app_v3/fetch_video_statistics`(单)/ `fetch_multi_video_statistics`(批) | $0.001 / $0.025 | 唯一有非零 play_count 的地方;返回字段有值才带 key,容错解析 |
| 评论 top100(hottest SOP) | `web/fetch_video_comments` count=100 翻 2-3 页 | ~$0.003 | 实收约请求数一半;无按赞排序参数(综合排序);app_v3 硬顶 20 条只当备用 |
| 弹幕(P1) | `web/fetch_one_video_danmaku` | $0.001/32s 窗 | 服务端固定 32s 窗口/8s 网格;`total` 一次给全量;9 分钟视频全量 $0.017 |
| 星图画像(P1) | `get_xingtu_kolid_by_sec_user_id` → `kol_audience_portrait_v1`+`kol_fans_portrait_v1`;打底 `xingtu_v2/get_author_base_info` | $0.001+$0.02×2;v2 $0.002 | 覆盖率=已入星图(4 万粉命中、1 千粉无),必须有"暂无星图数据"降级 |
| 热点/趋势/达人(P1) | `billboard/fetch_hot_rise_list`+`fetch_hot_category_list`;`index/fetch_multi_keyword_hot_trend`(先 `fetch_all_valid_date`);`billboard/fetch_hot_account_list` | $0.001-0.003 | hot_account 的 `query_tag` 必须传对象;两套垂类 id 空间(content_tag ≠ 热榜 sentence_tag);不碰 hot_words/creator 族/hot_calendar |

**明确不用**:yt-dlp(不需要)、代理池(不需要)、$0.005 高清播放链(play_addr 已是无水印源)、web 版 fetch_user_post_videos。

## 2. 数据模型与迁移

**迁移 0033**(独立一条,不带 DML——PG 新 enum 值同事务不可用):

```sql
ALTER TYPE "public"."platform" ADD VALUE 'douyin';
```

pgEnum 唯一定义在 `packages/db/src/schema/channels.ts:5`,四张表(channels/competitor_accounts/own_accounts/projects)自动生效。**零新表、零新列**:

- `clerk_videos.content_type` 是自由文本列,新增取值 `douyin_video` / `douyin_image`,不用迁移
- `transcript_source` 同理:`douyin_asr` / `douyin_text`
- `muse_monitor_videos` 无 platform 列(join competitor_accounts 得知),零改动
- P1 弹幕/画像数据:弹幕曲线作为分析中间产物进 prompt,不落库(重跑成本 $0.017 可接受);星图画像若做缓存再议(P1 设计时定,可能加 jsonb 列到 competitor_accounts)

**platformKey = `sec_user_id`**,正则 `^MS4wLjABAAAA[A-Za-z0-9_-]{43,64}$`(89 样本实测仅 55/76 两档全长)。跨搜索/profile/分享链一致,可从主页 URL 直接正则抽取,无 YouTube 式 needsResolution 二段解析(短链先展开即可)。

**contentType 决策:用独立值 `douyin_video`/`douyin_image`**(不复用通用 "video")。理由:SOP 引用标签需要平台出处("抖音视频"vs"YouTube 视频")、图文管线需要判别。**代价是必须同步改 4 处下游**(触点盘点抓到的陷阱):
- `packages/prompts/src/clerk.ts:115,212` 的 `isVideo = contentType === 'video'` 严格等于
- `apps/worker/trigger/analyze-channel.ts:1375` mapContentType 的归一化
- `analyze-single-video.ts:71` 图文拒绝条件扩为 `xhs_image || douyin_image`
- content-type/transcript-source badge 的两处重复定义(顺手合并成单一来源)

## 3. integrations 层(净新增)

`packages/integrations/src/clients/douyin.ts`,照 xhs.ts 结构,导出:

- `expandDouyinShortLink(input)` — v.douyin.com 302 跟随,10s 超时,失败返回原输入(镜像 expandXhsShortLink)
- `extractDouyinSecUserId(url)` / `extractDouyinAwemeId(url)` — 纯正则,主页 URL / video URL / iesdouyin share URL 全覆盖
- `resolveDouyinUser(profileUrlOrId)` — v1 profile,归一化 `DouyinUser {secUserId, uid, uniqueId, nickname, avatarUrl, followerCount, awemeCount, signature}`
- `getDouyinUserVideos(secUserId, limit)` — app_v3 列表 + 分页,归一化 `DouyinVideo`
- `getDouyinVideoDetail(awemeId | shareUrl)` — 详情(拆解时现取 play_addr/music/cover)
- `getDouyinVideoStats(awemeIds[])` — statistics 批量回填,容错解析
- `getDouyinTopComments(awemeId, n)` — 翻页凑 n 条,客户端按 digg_count 重排
- `computeDouyinEngagement(stats)` — `digg + 2×collect + 3×comment + 5×share`(与 XHS 同权重,跨平台口径一致)
- `pickJpegUrl(urlList)` — 从 url_list 扫 `.jpeg` 变体(**不能改后缀**,签名绑路径,改了 403;与 XHS 的 format 改写相反)
- `isOriginalSoundtrack(music, videoDurationMs)` — 原声守卫:标题含"原声"且 mp3 时长≈视频时长(±2s)

**get() 层修掉已知弱点**:AbortSignal 超时(现有 tikhub.ts/xhs.ts 都没有)、检查 body code、429 读 retry-after、区分三种错误形状(干净 status_code / 裸 `{"error":...}` / 静默回传)。限频 10 次/秒比 XHS 的 1 次/秒宽松,调用间 sleep 可降到 150ms。

`validators.ts` 加 `isValidDouyinProfileUrl` + `findDouyinShortLink`;`references.ts` 加 `fetchDouyinReference`(kind 联合类型扩三元)。

## 4. worker 管线

**analyze-channel.ts**:先把 `if(xhs)/else` 重构为显式 `if(xhs)/else if(youtube)/else if(douyin)`(douyin 落进隐式 youtube 分支是现在的必然事故),再写 douyin 分支:

- 选片:urls(短链展开+extractDouyinAwemeId,间隔 150ms)/ newest / popular(拉 limit×4 按 engagement 排序,同 XHS);recency 过滤可支持(列表带 create_time,顺手放开 UI 的 YouTube-only 门)
- 转写(ASR 双路径):
  1. 快路径:`isOriginalSoundtrack` 命中 → music.play_url(完整 MP3)直接走现有音频 ASR(qwenFirst,中文内容)
  2. 兜底:最低码率 H.265 下载(~4.2MB/min)→ ffmpeg -vn 抽 AAC(0.39MB/min)→ ASR
  3. 图文:desc 全文当正文(`douyin_text`),同 XHS 图文
  - 时间戳:走 Qwen 时无精确词级时间轴 → prompt 用近似时间规则(见 §6);无官方字幕,不存在 caption-first
- vision:封面 `pickJpegUrl(cover.url_list)`;图文最多 9 张进 analyzeImageStack(同 XHS);图片 URL 可落库(14-30 天有效),重分析时刷新
- 落库:`title = desc 截 80 字`(纯展示),desc 全文进 transcript 头部;`views = engagementScore`;**Top-K(views 排序前 3)+ hottest 候选用 `getDouyinVideoStats` 回填真实 play_count**(一次 multi 调用 $0.025 或逐条 $0.001×K)
- hottest 评论:`getDouyinTopComments(top1, 100)`(YouTube 之外第二个有评论摘要的平台;XHS 仍无)

**monitor-competitors.ts**:同构三路分支;douyin 对标走 `getDouyinUserVideos`;`xhsContentType` 过滤字段改名 `contentFilter` 同时作用于 xhs+douyin 对标(旧字段名兼容读一版)。

**generate-script.ts:128** 顺手修:Muse 引用块硬编码 `type:"youtube"`,改为按来源平台(现在 XHS 来源也被错标)。

**detect-channel-series**:维持显式拒绝(douyin 有官方合集数据,P1 用 mix_detail/mix_post_list 直接出系列,免 LLM 聚类;注意 episode 是倒序编号,按 create_time 重排)。

## 5. web 层

- 添加账号:create-channel-form / new-account-sheet / edit-channel-sheet 的 Select 加抖音项 + placeholder(`https://www.douyin.com/user/...` 或 v.douyin.com 短链)

- muse-start-sheet:hasDouyin、contentNoun 文案、内容类型选择器对 douyin 也显示(contentFilter)

## 6. prompts 层

- 新增 `DOUYIN_VIDEO_PREAMBLE` / `DOUYIN_IMAGE_PREAMBLE`(镜像 XHS 版):声明平台语境、views=加权互动分口径(回填过真实播放数的注明)、时间规则用近似段落引用(Qwen 路径无精确时间轴);图文版同 XHS 图文规则
- `renderVideoSummaryBlock` 引用标签加"抖音视频/抖音图文"分支;上线后按 v0.8 封面修复的方法(冻结输入+N 次采样)复验引用格式遵从率
- muse.ts / poet.ts / clerk-comments.ts 平台无关,零改动(盘点确认)

## 7. 计费与成本

- 配额口径:douyin 视频=实际分钟(同 YT/XHS 视频),图文=5 分钟 flat(同 XHS 图文)——沿用现有 stamp 逻辑,无新规则
- TikHub 成本(20 条频道分析):列表 $0.001 + 详情 20×$0.001 + 统计回填 $0.025 + 评论 $0.003 ≈ **$0.05/run**;ASR/vision 与 XHS 同价(仍是大头);无代理流量成本(对比 YouTube 每 run 的住宅代理流量,抖音单位成本显著更低)
- P1 增量:弹幕 $0.017/条(只对 hottest/单视频拆解)、星图画像 $0.041/账号(一次性,可缓存)、热点巡视 $0.002/次

## 8. 分期与排期

**P0 平价接入**(热点 top5:analyze-channel 管线 > douyin client > routers 三分支 > monitor-competitors > UI 面):
1. 迁移 0033 + douyin.ts client + validators + domain key(可独立先行,纯新增)
2. worker 两管线三路重构 + douyin 分支 + references + 评论
3. routers + schemas + UI 全触点
4. 测试(见 §9)→ 部署(迁移先行 → push Vercel + Trigger.dev deploy 同批)

**P1 抖音特色**(P0 稳定后,每项独立可发):
1. 弹幕留人曲线:单视频拆解/hottest 注入密度曲线+峰值弹幕文本(片头打卡噪音降权),RETENTION 章节从推测变实证
2. 星图受众画像:对标详情卡 + Bible AUDIENCE 生成时注入(有 kolid 才注,无则现状);v2 主题标签 $0.002 打底
3. Muse 平台热点巡视模式 + 选题趋势徽标(hot_rise + multi_keyword_trend)
4. 垂类达人推荐(hot_account_list → 一键导入对标)
5. 官方合集 → 系列面板(mix 数据,免 LLM)

## 10. 风险与预案

| 风险 | 预案 |
|---|---|
| TikHub 静默弃用(XHS 已两次) | client 对"空列表但 has_more 异常"记 warn;运维靠 usage_events 观测调用成功率 |
| app_v3 与 web 家族数字口径差 24% | 粉丝数固定走 web v1,列表/统计固定走 app_v3,永不混线 |
| 无效 id 返回 200 且计费 | client 层判 `comments:null`/`user:{}` 等退化形状,当失败处理 |
| Qwen 路径无精确时间戳 | prompt 近似时间规则(XHS 已验证可行);P1 可试 Deepgram Nova-3 中文+词级时间戳升级 |
| 星图覆盖率 | UI 显式"暂无星图数据"降级,不阻塞主流程 |
| 迁移枚举值 | 0033 独立无 DML;先上迁移再发代码,旧代码遇 douyin 值前 UI 还选不到,无窗口风险 |
