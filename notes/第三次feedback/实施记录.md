承上节「M4 = 上游 source 数据错」的结论，做 source 层事实核查 + citation。用户决策：**标记存疑+给建议、不自动改** / **仅 Poet 选题分析** / **行内「来源+状态」badge**。

**实现**：
- **DB**：`poet_custom_topics` 加 `fact_checks` jsonb（`CheckedFact = {fact, src, status: verified|disputed|unsupported, note?}`）；迁移 `0011_add_fact_checks`（additive，已应用 prod）。
- **服务**：`services/poet/factCheck.ts` `factCheckVerbatim` —— 解析 verbatim 的 `[src:]` 行 → LLM（Pro 世界知识）逐条判 status + note；**保守**（不确定→verified）、**全 fallback**（失败/截断/解析不了→全 verified，绝不阻断、不误伤）。prompt `buildFactCheckPrompt`。
- **集成**：`analyzeTopic` 内聚核查（grounding 后）→ 返回 `factChecks`，`analyze-custom-topic` 存库；写稿 `formatVerbatimFacts` 对 `disputed` 行注入 `⚠️ DISPUTED — <note>; do NOT state as fact`（让 writer 泛化，错误年份不进终稿）。
- **前端**：`PoetFactList`（行内「来源」badge，有 url 则 badge 即链接 + hover tooltip 显标题/url/片段；`✓已核实`/`⚠存疑` 状态 badge；存疑展开 note 建议）+ Badge `success`/`warning` 变体；topic 详情页 + poet 主页两处。**栈是 base-ui（非 radix）**，故链接放 trigger、tooltip 纯展示，零新依赖。

**真机验证（真 LLM）**：
- `factcheck-smoke`（`analyzeTopic` 全链路，77s）：10 facts → **8 verified / 2 disputed**。M4 1964→disputed（note「公认 1967」）；120 中画幅→disputed（note「实为 135，1/1000s 属实」）；M3 0.91x / M7 DX / M5 / M6 等 8 条对的全 verified。**precision / recall / 不误报 = 100%**。
- `factcheck-script-smoke`（`writeScript` short+grounding，98.5s）：终稿提 M4 但**不含「1964」**（泛化为「快速装片」），M3/M6 真实年份保留。写稿闭环生效。

**受限**：prod 既有 topic 的 `references` 为空（历史数据问题），且 prod DB 写（UPDATE/INSERT topic）被安全层拦，故端到端走 `analyzeTopic` 纯函数 smoke（等价完整链路）。前端 FactList 需一个经新版 analyze 且有 references 的 topic 才显示 —— 在 UI 新建 custom topic 贴素材分析即可看到。
