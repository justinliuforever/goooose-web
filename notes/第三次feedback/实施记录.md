## 与计划的偏离（有据）

1. **C1 JSON 模式没上**。计划说「先 smoke 验证 @ai-sdk/deepseek 透传 response_format；不透传则退回 jsonrepair」。实测：deepseek-v4-flash 在简单 prompt 上认 json_object，但在真实 15-key 分析 prompt 上 **json 模式更慢（41s vs 33s）、可靠性无增益（两者经 lenient parse 都 15/15）、且首跑 json 反而 strict 解析失败**。生产「Could not parse」根因是 **Pro 推理烧空预算→空输出**，已由「分析切 Flash」根治。故 **不引入 response_format**，保留 lenient parseAnalysis + 每条 retry。`llmJson` 辅助已撤回。
2. **C2 保守**。#1 wrapper 已 A/B 验证能去生造词（无需改 inner），故 C2 只删两段明显冗余的 NEW 子节，不大改 SOP 结构（避免破坏调好的产出）。
3. **#6 用 expand 而非 rename**。新增 `duration_seconds` 并回填，保留 `duration_minutes`（旧部署代码仍可读，零中断、可回滚）；drop 旧列留到 contract 步（Part 2 清理）。
4. **C6 强插品牌口头禅 = 移除**（已获批「测试后用数据决定」）。数据：11 份激活 Bible 中 9 份会被 anyQuoted 兜底硬塞**非品牌**的随机引号短语进开头钩子。结论：**移除强插是对的**，保留 prompt 内软提示即可。
