# Release v0.6 — 准入与配额体系 + 圣经文件导入(阶段性收敛)

汇总 Round 5(ACL/计量/管理后台)与 Round 6(圣经文件导入与架构升级)两轮工作。两轮同属 v0.6 版本线;本文档取代 round5-*/round6-* 系列文档(研究过程与审计明细已收敛,过程性内容不再保留)。v0.6 未完结,本文为阶段性收敛快照(2026-07-07)。

## 一、功能清单

### 准入与配额(Round 5)
- **邮箱准入(ACL)**:应用层白名单(Logto 保持开放注册,授权在自家 DB)。未授权邮箱登录后进「申请内测」页;管理员审批通过自动放行(审批邮件通道已建,待域名激活 Resend)
- **用量计量**:AsyncLocalStorage 上下文 + AI SDK 中间件在 `llm()` 工厂单点捕获,LLM token/ASR 秒/TikHub 调用/vision 全记 `usage_events`,按用户/run/feature 归属
- **配额:单一分钟池** 300 分钟/月(Asia/Shanghai 月初重置)。计费规则:视频按实际分钟(≥1)、图文 5、写稿按目标时长(≥2)、圣经 5、**文件导入圣经 10**、选题 3、单视频 SOP 2。生成类触发时扣,分析类结束按实结算
- **失败退款**(R6 补):任务失败/取消/被回收时原子幂等退回已扣分钟(`quota_charged`/`quota_refunded`),四条失败路径全接,生产实测两次精确退回
- **兑换码**:面额=当月分钟,月底随池重置;账号隐形上限 30
- **管理后台 /admin**(仅 admin 角色):套餐与额度规则卡、准入审批、预邀请白名单、兑换码(含使用者)、用户管理(角色下拉/停用/删除/详情侧板含 token/登录/IP)、用量总览
- **版本徽章**:页面显示当前 Beta 版本

### 圣经文件导入(Round 6)
- **入口**:生成圣经弹窗新增「导入文件」tab,拖拽/点选 md/txt/pdf/docx,≤15MB
- **上传**:浏览器算 sha256 → 2MB 分片走 tRPC 入库(幂等可重试)→ 服务端重组校验 sha256+文件魔数 → 才触发任务、才扣额度。全程自有域名(大陆访问约束,不引入第三方浏览器直连端点)
- **提取(按风险分路)**:
  - 数字 PDF:Anthropic 原生 PDF 输入(≤15 页单次调用;>15 页 pdf-lib 切块+1 页重叠;>40 页截断提示)。转写后与 PDF 自带文本层做数字集交叉核对(确定性、零成本)
  - 扫描件/图片:纯视觉转写 + 第二次调用只核数字(verify-pass)
  - docx:mammoth 提正文按序落图片占位 + 与 zip 媒体清单对账(防 WPS 浮动图丢失)→ 内嵌表格截图逐张视觉转写回填;检测在线表格裁切预览并提示补充完整版
  - 全程逐字纪律指令(温度 0,看不清标「无法辨识」不猜);CJK 部首码位/全角数字归一化
- **生成**:转写全文 → DeepSeek Pro 重构为锚点圣经(≤7000 字)→ grounding(source=转写全文)→ **构建时数字审计**(圣经数字 ⊆ 转写,违规重生成→再败删行标记)
- **逐字段确认门**:任何存疑项(无法辨识/截断/审计违规/图转写失败)→ 圣经保存但不激活;确认卡逐条 resolve,服务端硬门:未确认完不给激活
- **失败出口**:垃圾文件魔数拒(扣费前)、空白/加密件明确中文报错中止+退款、弃传 2h GC、失败孤儿分片 24h GC

### 圣经架构升级(手动/导入统一)
- **锚点化格式**:`TOPIC:`/`HOST:` 行 + 9 个英文锚点章节(POSITIONING/PERSONA/AUDIENCE/CONTENT_PILLARS/CONTENT_RULES/METHODOLOGY/INFORMATION_SOURCES/TOPIC_FRAMEWORK/FACT_SHEET)。存量旧圣经已 LLM 转换(15/19 成功,3 份双向数字审计从严拒绝保持旧格式,回退机制兜底零损失,转换前全表备份)
- **按需取节**(`selectBibleSections`,无锚点回退整块):
  - 写稿 prompt:POSITIONING+PERSONA+CONTENT_RULES+METHODOLOGY;grounding source 另加 FACT_SHEET+转写全文(导入圣经)
  - 选题分析:定位类六节,**不给事实节**(其 prompt 硬规则禁止从圣经取事实);导入圣经的 METHODOLOGY+FACT_SHEET(均过数字审计)走独立「已核实账号事实」块,且计入其 grounding source
  - Muse:定位摘要三节(约 900 字)
- **身份门**:`HOST:` 行进 identityRule/scrubForeignSelfIntro allowlist 并**优先用于自称**——导入人设的"我是徐艳梅"正确保留,不被身份清洗误伤;手动圣经无 HOST 行为不变

## 二、质量验证(全部真实 API/生产实测)

- **提取保真**(真样本 bake-off):7 页嵌套表格 PDF 单次转写零幻觉零遗漏(坐标/韧带清单/剂量/19 人名逐项对);模拟扫描件(100dpi)数字保真 100%;docx 31 张表格截图数据精确、裁切处如实标注不编造
- **技术选型证据**:FinCriticalED 基准 Claude Sonnet 4.6 中文文档事实保真第一(97.23%);Sonnet 5 对照实验输出 391 处部首脏码位+不支持 temperature,维持 4.6
- **护栏实战**:数字审计真实抓到一次生成阶段编造并删行;grounding 植入 3 处错误 9/9 全抓零误伤;纵深防御实测(上游分析讲反术语,终稿自动纠正)
- **独立严审 + 三轮修复闭环**:Opus 评审逐条主张对照原文档,发现署名用频道名、选题分析反转客户签名术语(真/假韧带)、脚本编造注射坐标+步骤倒置;三轮修复(HOST 优先自称/grounding 加解剖与步骤序 claim/已核实事实块——根因是分析后置 grounding 空源泛化)后生产复测**全绿**:术语/剂量/禁忌/四步顺序与原文档逐字一致
- **Edge cases 生产实测**:md 导入(归一化/HOST/不越权激活)、损坏 docx(失败+精确退款)、空白 PDF(中止+退款)、Muse 巡视(6 选题贴合定位、事实带"需核实")、配额扣/退/结算全对账
- **Beta 就绪体检**(R5,3 Opus 深审):2 个 P0 已修——RLS 全表锁定+撤销 anon/authenticated(此前 anon key 可互联网读写整库,实测确认后修复验证 401)、长任务 maxAttempts 3→1(消除重试风暴+重复扣费);维护 cron×3(卡死 run 回收/代理重启/导入 GC)
